# InstaDM for Windows — Agent Operating Manual

Every agent/model session working in this repository MUST follow this document.

## What this project is

A native-feeling, privacy-absolute Windows application inspired by InstaDM (macOS):
a focused Instagram **direct-messages-only** client that embeds Instagram's own
authenticated web experience, contains all distracting Instagram surfaces (feed,
Explore, Reels, Stories, profiles), and never sends any application-owned data
anywhere except Instagram/Meta itself.

## Session resume procedure (mandatory, in order)

1. Read this file and `.cursor/rules/instadm-windows.mdc`.
2. Read `docs/HANDOFF.md`, `docs/IMPLEMENTATION_PLAN.md`, `docs/DECISIONS.md`,
   and `docs/PRIVACY_THREAT_MODEL.md`.
3. Resolve the authorized Obsidian Windows notes directory from the ignored
   `.instadm-local.env` (variable `INSTADM_WINDOWS_NOTES_DIR`). Read
   `HANDOFF.md` and `Implementation Progress.md` there. Verify the two handoff
   copies match.
4. Run `git status --short --branch`, inspect recent commits and any diff.
5. Repository state wins over stale prose. Repair both handoff copies
   immediately if they disagree with reality.
6. If the prior session may have been interrupted, re-run the smallest relevant
   build/test command before new work.
7. Continue the exact `Next action` from the handoff unless repository evidence
   makes it unsafe.
8. Update and re-synchronize both handoff copies before and after new work.

## ABSOLUTE PRIVACY INVARIANT (never violate)

All application-owned processing, settings, state, diagnostics, and storage stay
local to the user's Windows machine. The only permitted network traffic from
app-owned processes is first-party Instagram/Meta traffic generated by the
embedded official Instagram web client.

Concretely, NEVER (in code, tests, CI, docs, handoffs, or commits):

- Capture, read, store, log, serialize, or transmit credentials or 2FA codes.
- Read, retain, display, log, or serialize cookie values, tokens, auth headers,
  or CSRF values. Native code may check *existence* of a named session cookie
  only, never its value.
- Send messages, previews, sender names, usernames, media, DOM text,
  screenshots, or notification content to any analytics, telemetry, AI, cloud,
  crash-reporting, or developer service.
- Route Instagram traffic through any proxy, relay, developer server, or
  unofficial API.
- Add analytics, telemetry, remote config, cloud sync, push backend, remote
  logging, automatic crash upload, or an app-controlled update server.
- Commit real credentials, sessions, cookies, messages, screenshots, packet
  captures, personal absolute paths, or private note text.
- Enable remote debugging or DevTools in Release builds.
- Print or `cat` the ignored `.instadm-local.env` or WebView user-data files.

Development-time boundary: do not open WebView user-data folders, cookie
databases, live-account logs, or captures. Use synthetic fixtures for all
model-assisted debugging. Runtime network audits expose only sanitized
destination metadata, never payloads or private URLs.

## Read-only reference material

- The macOS source app (path in `.instadm-local.env`, `INSTADM_MAC_SOURCE_DIR`)
  and every Obsidian note outside the `Windows App/` subfolder are strictly
  read-only. Never modify, rename, move, or delete them.
- Only the `Windows App/` Obsidian subtree may be created/updated, and only with
  sanitized content safe for iCloud sync.
- Never copy private notes wholesale into this repository or into prompts.

## Working rules

- Keep the repository buildable at every commit. Atomic commits at meaningful
  checkpoints; run tests and the privacy scan (`tools/`) before committing.
- Pure policy/state-machine logic lives in `src/InstaDM.Core` (cross-platform,
  unit-testable on any host). Windows UI/WebView2 integration lives in
  `src/InstaDM.App` (Windows-only; builds via Windows CI when the host is macOS).
- Never mark a milestone done if its required Windows build/test was not run;
  record `NOT RUN - WINDOWS ENVIRONMENT REQUIRED` honestly.
- Tests never use a live Instagram account, real DOM, or real session data.
- The GitHub repository stays **private** unless the user explicitly says otherwise.
- Update `docs/HANDOFF.md` first, then mirror the identical sanitized body to
  `Windows App/HANDOFF.md`, then verify they match (`tools/sync-handoff.sh`).

## Task delegation

Tasks in `docs/IMPLEMENTATION_PLAN.md` are marked whether they are safe for a
smaller fallback model. Smaller models: only take tasks marked
`FALLBACK-SAFE: yes`, follow the acceptance criteria exactly, and never make
undocumented architectural or privacy decisions.
